- # Building Tutor RAG: A Next-Generation AI Teaching Assistant Powered by Groq, Pinecone, and FastAPI
  
  The landscape of education is shifting. Traditional learning management systems (LMS) offer static content—PDFs, slides, and recorded lectures. But what if students could converse with their course materials? What if teachers could upload a syllabus and instantly generate tailored quizzes? 
  
  In this post, we will walk through the architecture and implementation of **Tutor RAG**, a comprehensive AI-powered teaching assistant. Tutor RAG enables teachers to upload educational PDFs and allows students to ask questions, receive context-aware answers, and generate interactive quizzes based on the uploaded material.
  
  We will explore the complete stack, from the robust **FastAPI** backend and **MongoDB** database to the blazing-fast **Groq** LLM inference, **Pinecone** vector database, and the premium vanilla **HTML/CSS/JS** frontend.
  
  ---
- ## 1. System Architecture Overview
  
  At its core, Tutor RAG is an implementation of **Retrieval-Augmented Generation (RAG)**. RAG is a technique that grounds large language models (LLMs) in external knowledge sources, preventing hallucinations and ensuring that the AI acts as a strict tutor rather than a creative novelist.
  
  > [!TIP]
  > **Why RAG for Education?** Hallucinations in education are dangerous. A student learning biology cannot afford an AI that invents facts about cellular respiration. RAG ensures the AI only answers based on the teacher's provided materials.

- ## 2. The Backend: FastAPI & MongoDB
  
  We chose **FastAPI** for the backend due to its speed, asynchronous support, and automatic API documentation.
- ### Role-Based Access Control
  The platform serves two distinct user roles: **Teachers** and **Students**. We manage user data, chat histories, and quiz results using **MongoDB**.
  
  When a user logs in, the `/login` endpoint authenticates them using HTTP Basic Auth (with hashed passwords) and returns their context so the frontend can route them appropriately:
  
  ```python
  @router.get("/login")
  async def login(user=Depends(authenticate_user)):
    return {
        "message": "Login successful",
        "user": {
            "id": str(user["_id"]),
            "username": user["username"],
            "fullname": user["fullname"],
            "role": user["role"]
        }
    }
  ```
- ### Tracking Progress
  Every interaction is logged. When a student takes a quiz, the result is saved in the `quiz_history_collection`. This allows students to track their progress over time and see their percentage scores improve.

- ## 3. The RAG Pipeline: Langchain, Gemini Embeddings, and Pinecone
  
  The "Retrieval" in RAG relies on vector mathematics. When a teacher uploads a PDF, the document is split into smaller, manageable chunks.
  
  To convert text into vectors, we use **Google Generative AI Embeddings** (`gemini-embedding-001`). These embeddings are incredibly rich, capturing the semantic meaning of educational text.
  
  These vectors are then upserted into **Pinecone**, a managed vector database that provides ultra-fast similarity search.
  
  ```python
  # Embedding and Vector Search Setup
  pc = Pinecone(api_key=PINECONE_API_KEY)
  index = pc.Index(PINECONE_INDEX_NAME)
  
  embed_model = GoogleGenerativeAIEmbeddings(model="gemini-embedding-001")
  ```
  
  When a student asks a question, the system embeds the question using the same Gemini model, queries Pinecone for the top 5 most similar document chunks, and retrieves the original text from MongoDB using the chunk IDs.

- ## 4. The Brain: Groq and LLaMA 3.3
  
  Once the context is retrieved, we need an LLM to formulate the final answer or generate the quiz. We selected **Groq**, specifically running the `llama-3.3-70b-versatile` model. 
  
  > [!IMPORTANT]
  > **Why Groq?** Groq utilizes LPUs (Language Processing Units) rather than GPUs, resulting in inference speeds that feel instantaneous. This is critical for an interactive chat application where students expect real-time feedback.
  
  We manage the prompts using **Langchain**. Here is the prompt used for answering student questions:
  
  ```python
  prompt = PromptTemplate.from_template(
    """
    You are a helpful assistant.
    Use the following context to answer the question.
    If the answer is not in the context, say so.
    
    Context: {context}
    Question: {question}
    Answer:     
    """
  )
  rag_chain = prompt | llm
  ```
  
  This strict prompt ensures the AI does not stray outside the bounds of the provided context, maintaining educational integrity.

- ## 5. Deep Dive: Dynamic Quiz Generation
  
  One of the standout features of Tutor RAG is its ability to generate multiple-choice quizzes on the fly. 
  
  The student simply requests a topic (e.g., "Photosynthesis") and a number of questions. The system performs a vector search on the topic to gather relevant context, and passes it to Groq with the following prompt:
  
  ```python
  quiz_prompt = PromptTemplate.from_template(
    """
    You are a test-generator assistant.
    use context below to generate {num_questions} multiple choice questions 
    based on the following context.
    
    Context: {context}
    
    For each question, provide:
    1. The question text.
    2. Four options (A, B, C, D).
    3. The correct answer.
    """
  )
  ```
- ### The Challenge of Parsing LLM Output
  LLMs output raw markdown text. Initially, we just displayed the raw text to the user. However, this meant the "Correct Answer" was visible immediately, defeating the purpose of a quiz!
  
  To solve this, we built a custom parser in JavaScript on the frontend. The parser iterates through the markdown line-by-line, identifies the questions, strips out the "Correct answer" lines, and dynamically maps the options (A, B, C, D) into interactive HTML radio buttons.
  
  ```javascript
  // A snippet of the frontend parsing logic
  if (line.match(/^(?:\*\*)?Question\s*\d+/i)) {
    // Start a new question object
    currentQuestion = { text: line, options: [] };
  } 
  else if (line.match(/^(?:\*\*)?Correct answer/i)) {
    // Skip this line to hide the answer from the student
    continue;
  }
  else if (line.match(/^([A-D]|[1-4])[\.\)]/i)) {
    // Extract the option letter and text, then push to the options array

  ```
  
  This transforms a raw text block into a beautiful, interactive, clickable quiz card.

- ## 6. The Frontend Interface: Premium Vanilla Web Design
  
  In an era of heavily bloated React and Next.js applications, we opted to build the Tutor RAG frontend using purely **Vanilla HTML, CSS, and JavaScript**. 
  
  Why? Because with the power of modern CSS variables, Flexbox, and minimal DOM manipulation, you can create breathtaking, performant applications without a build step.
- ### Key UI Features
- **Glassmorphism & Dark Mode**: The UI features a sleek `#0f172a` primary background with deep slate panels, making it easy on the eyes for late-night studying.
- **Unified Dashboards**: Both Teachers and Students have access to a split-screen dashboard. On the left is the RAG Chat Interface, and on the right is a tabbed Tools Panel.
- **Teacher Tools**: Teachers have an "Upload" tab where they can drag-and-drop PDFs to instantly expand the global knowledge base. They also have access to the chat and quiz tools to test the system exactly as a student would experience it.
- **Markdown Parsing**: We integrated the `marked.js` library so that the AI's chat responses—including math formulas, bold text, and code snippets—render beautifully in the browser.
- ### The Interactive Quiz Player
  When a student generates a quiz, the Tools Panel seamlessly transitions from the "Generator" view to the "Player" view. 
  
  Instead of typing answers into a text box, the parsed radio buttons appear. We styled these radio buttons to act as large, clickable cards that shift slightly on hover (`transform: translateX(4px)`), providing a highly tactile and premium feel.
  
  Once submitted, the backend verifies the answers, calculates a percentage score, and the UI immediately displays a color-coded breakdown of what the student got right or wrong.
